# M2.3 exact cosine reference implementation

**Status:** Implemented M2.3 reference path.  
**Governing decisions:** EDS v2.1.1 section 12; ADR-010, ADR-011, ADR-016;
accepted OQ-11; controlled `query-result/0.1.1` correction.  
**Input boundary:** the authoritative corrected M1 Draft successor package,
opened only as an immutable `VerifiedRelease`.

## Scope

`ExactCosineQueryCore` is the sole M2.3 executor. It accepts raw
`query-request/0.1.0` JSON, obtains the canonical request digest through the
M2.1 canonicalizer, and performs exact float32 inner products only after the
trusted local release boundary has verified and snapshotted the package.

The executor requires a manifest-addressed profile declaration and vector
shard with matching profile identity, version, dimension, and dtype. The
declared representation must be L2-normalized `float32`, with cosine and
`higher-is-more-similar` direction. It checks every scored vector is finite
and normalized; it does not infer compatibility from dimension alone.

## Result and ordering

The executor produces `query-result/0.1.1` success envelopes. Each result
binds the ReleaseManifest digest, canonical request digest, profile ID and
version/digest, cosine direction, `exact` mode, vector-shard digest, and the
fixed `score-desc-record-id-asc-v1` ordering policy. Scores are float32 inner
products and ties sort by stable record ID.

`query-result/0.1.0` remains an immutable historical contract record. Its
controlled successor supplies the ADR-011 provenance that M2.3 now emits.

## Explicitly deferred

M2.3 does not implement filters, traversal, cursor execution, caching, ANN or
FAISS, cross-profile search, SDK, REST, Explorer, benchmark measurement,
retrieval-quality claims, or public-release behavior. Valid requests requiring
the deferred filter, traversal, or cursor features receive a typed error;
there is no fallback or silent weakening of request semantics.

## Conformance evidence

`tests/contract/test_exact_cosine_query.py` creates a corrected M1 successor
fixture, opens it through the M2.2 reader boundary, validates the returned
result against `query-result/0.1.1`, checks profile/shard provenance and
deterministic request equivalence, and exercises the typed rejection paths.
The repository's locked contract suite is the acceptance command.
