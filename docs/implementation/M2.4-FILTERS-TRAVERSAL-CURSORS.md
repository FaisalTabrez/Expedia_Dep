# M2.4 Core filters, traversal selectors, and cursors

**Status:** Implemented M2.4 Core behavior.  
**Governing decisions:** EDS v2.1.1 §12.3–12.4; ADR-016; accepted OQ-11;
controlled `filter-expression/0.1.1` correction.  
**Input boundary:** immutable `VerifiedRelease` snapshots only.

## Filter behavior

Query Core evaluates corrected `filter-expression/0.1.1` expressions over the
five declared canonical M1 record fields. Boolean, equality, membership, and
four-state filtering run before exact ranking. `false` is distinct from
`present`, `missing`, and `unavailable`; no coercion or client-side fallback is
permitted.

The corrected M1 Draft does not declare annotation assertions, numeric
canonical fields with unit contracts, or DerivedRelation artifacts. Annotation
and range filters therefore return `unsupported_filter`; relation filters
return `unsupported_relation`. These forms are not interpreted as false and
are never silently removed.

## Traversal and cursors

Traversal selectors are validated with the M2 depth bound (default 1, maximum
2). This release has no declared relation artifact, so a valid selector returns
`unsupported_relation` rather than inventing an edge.

Results use opaque `cursor/0.1.0` payloads bound to the ReleaseManifest digest,
the canonical selection digest, ordering version, and last emitted score/record
key. The continuation token itself is excluded only from the selection digest;
changing release, profile, query record, filter, or ordering invalidates it.
Offset pagination is not implemented.

## Evidence boundary

Successful exact results retain `method_not_evaluated`, reflecting that no
retrieval-method evaluation has been performed. This is an evidence label, not
a quality, recall, performance, scalability, biological, or public-release
claim.

`tests/contract/test_m2_4_query_core.py` covers canonical filter application,
false-state distinction, typed unsupported forms, opaque continuation, changed
filter invalidation, and relation traversal refusal. The locked contract suite
is the acceptance command.
