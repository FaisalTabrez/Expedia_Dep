# EXPEDIA Project Context

## Purpose

EXPEDIA is an open scientific platform for constructing, validating, publishing, and exploring versioned genomic embedding atlases.

The long-term contribution of EXPEDIA is NOT the desktop application.

The primary scientific contribution is the publication of reproducible, versioned, citable genomic embedding atlases.

The desktop application, Python SDK, and APIs are clients of the atlas.

---

## Project Philosophy

This repository prioritizes

- scientific correctness
- reproducibility
- modular architecture
- long-term maintainability
- extensibility
- evidence-based engineering

over

- feature count
- implementation speed
- premature optimization

---

## Guiding Principles

1. Atlas releases are immutable.

2. Every architectural decision should have an explicit rationale.

3. Every scientific claim should be referenced.

4. Multiple embedding models must be supported.

5. Components should be loosely coupled.

6. Builder and Explorer are independent systems.

7. Validation is a first-class component.

8. The atlas is model-independent.

9. Every release must be reproducible.

10. The specification is the canonical source of truth.

---

## Current Status

Revision 3 represents the initial project report.

The goal is NOT to polish Revision 3.

The goal is to transform it into the official EXPEDIA Design Specification (EDS).

The EDS will become the implementation blueprint.

---

## Repository Philosophy

The repository follows documentation-first development.

Architecture is finalized before implementation.

Every implementation decision should trace back to the specification.

---

## Scientific Position

EXPEDIA is an infrastructure project rather than an annotation tool.

Its purpose is to organize genomic knowledge using embedding spaces generated by genomic foundation models.

---

## Expected Lifetime

This project is expected to evolve over multiple years.

Architectural stability is preferred over short-term convenience.
