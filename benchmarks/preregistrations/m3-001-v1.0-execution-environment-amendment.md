# M3-001 Version 1.0 controlled execution-environment amendment

**Status:** Accepted controlled amendment.  
**Study:** M3-001 Deterministic Exact Query Reproducibility, Version `1.0`.  
**Effective environment:** `EE-M3-001-v1.1`.  
**Date:** 2026-07-21.

## Triggering evidence

The originally approved executable,
`sha256:3c6a206b7d93cca823934a83732220dcffd413fd1036d9fb82eebb64599cf7f3`,
matches the initial manifest but cannot import `jsonschema`, a frozen
dependency required by the M1 Release Reader. M3.3.1 therefore failed before
release verification, query execution, or any replicate. The event is retained
in [`incident-log.json`](../../validation/evidence/m3-001/execution-v1/incident-log.json).

## Decision

The active execution binding is the immutable successor
[`m3-001-v1.1-evaluation-manifest.json`](../evaluation-manifests/m3-001-v1.1-evaluation-manifest.json).
It replaces only the executable identity with the lock-resolved project virtual
environment `.venv/Scripts/python.exe`, digest
`sha256:5912d0884b23c0343983a864c6064242391e2265536f50b88624857e353882c9`,
and identifies it as `EE-M3-001-v1.1`.

The original manifest and approval record remain immutable historical evidence.
This amendment is an execution-environment correction, not an ad hoc execution
change.

- research question, OQ-05 claim boundary, and prohibited claims;
- preregistered request corpus and three independent-process replicate design;
- `PASS`, `FAIL`, `INCONCLUSIVE`, and `ABORTED` definitions;

- M1 Draft release, profile, and vector-shard digests; and
- raw-evidence, incident, and retention requirements.

No scientific question, methodology, analysis, claim boundary, release
artifact, profile, Query Core behavior, or interpretation is changed.

## Acceptance criteria

- The successor manifest preserves every non-executable binding in logical
  value and differs only in the declared Python execution identity, its
  environment identifier, amendment/approval references, and supersession
  provenance.
- The new executable can import the lock-resolved `jsonschema==4.23.0`
  dependency.
- M3.3 restarts at environment verification; it does not reuse, replace, or
  reinterpret the retained failed pre-execution attempt.
