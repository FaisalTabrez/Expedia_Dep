# Evaluation manifests

Each accepted M3 study has one content-addressed evaluation manifest. The
manifest binds its approved preregistration, implementation checkpoint,
execution environment, controlled inputs, comparison rule, output location, and
claim boundary before execution begins. It is study evidence only; it neither
changes an Atlas Release nor defines a new Atlas contract.

[`m3-001-v1.1-evaluation-manifest.json`](m3-001-v1.1-evaluation-manifest.json)
is the active immutable manifest for approved M3-001 Version 1.0. The v1
manifest is preserved as historical evidence of the controlled executable
identity correction.

[`M3-002-EVALUATION-MANIFEST-TEMPLATE.json`](M3-002-EVALUATION-MANIFEST-TEMPLATE.json)
is retained as the historical planning template.

[`m3-002-v1-evaluation-manifest.json`](m3-002-v1-evaluation-manifest.json)
binds the approved M3-002 preregistration, frozen M2 checkpoint, declared
M1 v3 Draft identities, execution environment, reference-oracle source, and
oracle software-verification record. It is accepted as the immutable execution
definition for the preregistered M3-002 comparison; its separate maintainer
approval record binds the manifest digest. It authorizes no implementation or
methodology change and no interpretation or claim decision.
